8-Step Requirements Lifecycle NEW

SMART Requirements

  • Specific - Clear, unambiguous
  • Measurable - Quantifiable criteria
  • Achievable - Realistic to implement
  • Relevant - Tied to business need
  • Time-bound - Has deadline

MoSCoW Prioritization

  • Must Have - Critical, non-negotiable
  • Should Have - Important, workarounds exist
  • Could Have - Nice-to-have
  • Won't Have - Explicitly excluded

Elicitation Techniques NEW

  • Interviews, Workshops/JAD
  • Surveys, Document Analysis
  • Observation, Use Case Analysis

Key Principles

  • "Bake In, Don't Bolt On" - Security from start (retrofitting 10-100x more expensive)
  • Inside-Out = Start from assets → build security
  • Outside-In = Start from threats → work inward

WEEK 3: Security Architecture

Defense in Depth Layers

Perimeter → Network → Host → Application → Data → Admin → Physical

Redundancy vs Diversity

  • Redundancy = Same tech repeated (availability)
  • Diversity = Different tech each layer (attack resistance)

Network Segmentation

  • DMZ - Buffer between trusted/untrusted
  • VLANs - Layer 2 segmentation
  • Microsegmentation - Workload-level isolation
  • SDP NEW - Software Defined Perimeter (dynamic boundaries, resources invisible to unauthorized)

Firewall Types NEW

TypeLayerSpeed
Packet FilterL3-4Fast
StatefulL3-4 + stateMedium
App/ProxyL7Slow
NGFWAll + AI/TIVaries

IDS vs IPS NEW

  • IDS = Alerts only (passive)
  • IPS = Alerts + blocks (active)

3-2-1 Backup Rule NEW

3 copies, 2 media types, 1 offsite

Zero Trust (NIST 800-207)

"Never trust, always verify" - Assumes breach

7 Tenets: All resources protected, all comms secured, per-session access, dynamic policy, monitor all assets, strict auth before access, continuous improvement

ZTA Components

ComponentRole
PE (Policy Engine)Brain - makes grant/deny decisions
PA (Policy Admin)Establishes/shuts comms path
PEP (Policy Enforce)Gateway - enables/monitors connections

5 Verification Factors NEW

Who (identity), What (resource), Where (location), When (time), How (device posture)

Policy Hierarchy

Policies → Standards → Procedures → Guidelines
  • Policy = WHAT (mandatory, exec-approved)
  • Standard = HOW (technical specifics)
  • Procedure = Step-by-step instructions
  • Guideline = Recommendations (not mandatory)

Risk Responses

  • Avoid - Eliminate the activity
  • Transfer - Insurance, outsource
  • Mitigate - Reduce likelihood/impact
  • Accept - Document & proceed
Risk = Likelihood × Impact
Residual Risk = Risk after controls applied

Risk Appetite vs Tolerance

  • Appetite = How much risk org WANTS (strategic)
  • Tolerance = Acceptable VARIANCE (operational)

JIT Access NEW

Just-In-Time privilege elevation with automatic expiration

Impossible Travel NEW

Behavioral analytics flagging geographically impossible auth patterns

  2. Review Contextual Constraints

WEEK 7: Decision-Making

Trade-offs

  • Security vs Usability
  • Cost vs Coverage
  • Complexity vs Maintainability
  • Speed vs Thoroughness
  • Centralization vs Autonomy

ADRs (Architecture Decision Records)

Format: Title → Status → Context → Decision → Consequences → Alternatives

Decision-Making Traps NEW

  • Premature Convergence - Jumping to solutions too fast
  • Status Quo Bias - Baseline must be compared
  • Incomplete Info - Must decide before all facts known

Weighted Scoring Matrix NEW

Systematic evaluation: assign weights to criteria, score each option

Framework vs Mandate

  • Framework = Guidance, voluntary adoption (decentralized orgs)
  • Mandate = Policy enforcement, compliance required (regulated)

Decision Checklist

  • ✓ Stakeholders consulted?
  • ✓ Requirements prioritized (MoSCoW)?
  • ✓ Root cause found (Seven Whys)?
  • ✓ Options evaluated (4 dimensions)?
  • ✓ Trade-offs documented?
  • ✓ Decision defensible with evidence?
  • ✓ PoC conducted?
  • ✓ ADR created?
  • ✓ Rollback/exit plan exists?
  • ✓ Success metrics defined?

Case Study 1: EY Healthcare

Context

  • Cancer center + Level 1 trauma
  • 2M+ patients/year
  • 25,000+ managed devices
  • Extensive IoT medical devices
  • Legacy systems, can't patch (FDA)

Key Stats NEW

  • $11M avg healthcare breach cost (2023)
  • 53% increase over 3 years
  • 8 months to detect & resolve
  • 12%+ of annual spend on incidents
  • $80M+ revenue protected

Solution

  • Managed Threat Detection (EY)
  • Behavior-based EDR
  • Splunk (SIEM)
  • CrowdStrike (threat intel)
  • SOAR - automates 80% of routine responses NEW

Results

  • 50% increase in detection capability
  • Reduced MTTD & MTTR
  • Continuous monitoring of previously unmonitored devices

Lessons

  • Availability > Confidentiality in healthcare
  • Segment what you can't patch
  • SOAR essential at scale
  • Managed services solve talent gaps
  • "Partnership not abdication"

Case Study 2: Cimpress/Vistaprint

Context

  • $2B global company
  • 16,000+ employees, 20+ countries
  • 30-person security team
  • Decentralized subsidiaries
  • 100% cloud-native, NO internal network

Approach: Zero Trust

"Lead by Example, Not Mandate" NEW

3-Phase Implementation

  1. Phase 1 - Deploy at corporate (8-9K users)
  2. Phase 2 - Demonstrate value (COVID proof)
  3. Phase 3 - Gradual subsidiary rollout

Key Elements

Results

  • Seamless COVID WFH ("flip the switch")
  • Centralized visibility across subsidiaries
  • Fewer password tickets post-MFA

Lessons

  • ZT is a framework, NOT a product
  • Start with quick wins (MFA first)
  • COVID proved the architecture
  • Identity is the new perimeter
  • Cloud-native eliminates network dependency

Key Definitions

Core Concepts

  • Threat - Potential cause of unwanted incident
  • Vulnerability - Weakness that can be exploited
  • Risk - Likelihood × Impact
  • Control - Safeguard that reduces risk
  • Attack Surface - Total entry/exit points
  • Blast Radius - Scope of damage from breach

Principles

  • Least Privilege - Minimum access needed
  • Separation of Duties - No single person controls all
  • Need-to-Know - Access only for job function
  • Defense in Depth - Layered controls
  • Security by Design - Integrated from start
  • Assume Breach - Design for inevitable compromise

Detection & Response

  • EDR - Endpoint Detection & Response
  • XDR - Extended DR (cross-layer)
  • MDR - Managed DR (outsourced SOC)
  • SIEM - Security Info & Event Mgmt
  • SOAR - Security Orchestration, Automation & Response

Metrics

  • MTTD - Mean Time to Detect
  • MTTR - Mean Time to Respond/Recover
  • MTBF - Mean Time Between Failures
  • RPO - Recovery Point Objective (max data loss)
  • RTO - Recovery Time Objective (max downtime)

Auth & Access

  • RBAC - Role-Based Access Control
  • ABAC - Attribute-Based (most flexible)
  • MAC - Mandatory (gov/military labels)
  • DAC - Discretionary (owner controls)
  • SSO - Single Sign-On
  • SAML 2.0 - XML federation
  • OAuth 2.0 - Authorization
  • OIDC - Identity layer on OAuth

Exam Traps & Distinctions

FR vs NFR: "MFA in <5s 99.9%" = NFR (performance). "System must use MFA" = FR.
Zachman vs SABSA vs TOGAF: Zachman = taxonomy (NO process), SABSA = business-driven, TOGAF = ADM process
Engage vs D3FEND: Engage = ACTIVE/OFFENSIVE, D3FEND = PASSIVE/DEFENSIVE
ATT&CK vs Kill Chain: ATT&CK = NON-LINEAR (any tactic), Kill Chain = LINEAR (7 steps)
Redundancy vs Diversity: Redundancy = same tech (availability), Diversity = different tech (attack resistance)
Risk Appetite vs Tolerance: Appetite = how much WANTED (strategic), Tolerance = acceptable VARIANCE (operational)
Policy vs Standard vs Procedure: Policy = WHAT, Standard = HOW, Procedure = step-by-step
CISA ZTMM: Orgs must progress sequentially through maturity levels
