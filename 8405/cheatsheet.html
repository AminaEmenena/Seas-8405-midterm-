SEAS 8405 Finals — One-Page Cheat Sheet

W9: Enterprise Blueprints

  • Blueprint = set of related docs capturing design elements
  • Dual purpose: (1) capture future state (2) create buildable docs
  • TOGAF = Architecture Vision; SABSA = Conceptual Security Architecture
  • 4 reasons: Socialize Design, Implementation Template, Progress Baseline, Status Reporting
  • Doc principles: minimize shelfware, personnel continuity, collaboration enablement

TOGAF ADM

  • 5 advantages: Extensive docs, Free, Widely recognized, Customization (§4.3), Systematic
  • Phases: Preliminary → A (Vision) → B (Business) → C (Info Systems) → D (Technology)

Kill Chain & Assets

  • Lockheed Martin Kill Chain: Recon → Weaponization → Delivery → Exploitation → Installation → C2 → Actions on Objectives
  • Assets: data, personnel, IP, processes, reputation, trust (not just technical)
  • 3 Qs: What protecting? (Assets) What can go wrong? (Threats) Result? (Risk = Impact × Likelihood)

Goals, Roadmap & Vision

  • Goals (what) vs Capability (how) — need both
  • Roadmap 7 components: Charter, Scope, Principles, Goals/Outcomes, Transition Milestones, PM Artifacts, Impacts
  • Two-phase: Phase 1 = High-level Roadmap (validate first), Phase 2 = ADD (Architectural Definition Document)
  • Top-down (exec sponsor) vs Bottom-up (show value) vs Hybrid (preferred)
  • Architectural Principles: Name, Statement, Rationale, Implications
  • Vision 3 components: Principles, Scope, Target Future State
  • 4 Bootstrap Activities: Resource Discovery, Key Stakeholders, Validate Goals, Establish Feedback

W10: Technical Design & Execution

  • 3-Phase Journey: Conceptual Design (Ch 5-6) → Technical Design (Ch 7) → Execution

5-Step Technical Design

  • 1) Create Implementation Strategies
  • 2) Assess Synergies & Constraints
  • 3) Validate Against Threats
  • 4) Validate with Stakeholders
  • 5) Finalize Documentation

Key Concepts

  • Sound Wave Analogy: Constructive interference (strategies amplify) vs Destructive (strategies cancel out)
  • 6 Interrogatives: WHO, WHAT, WHEN, WHERE, WHY, HOW
  • MITRE ATT&CK for validation — use enterprise tactics to identify gaps
  • Healthy Friction (good: critical thinking, better decisions) vs Unhealthy Friction (bad: burdensome, false positives)
  • Skeleton Solution Documents: "straw man" designs, refined as you engage teams
  • Stakeholder Engagement: enable via formal detail, engage SMEs, validate technical feasibility

W11: Telemetry & Metrics

  • 4 reasons telemetry matters: Guide Decisions, Detect Early, Validate Effectiveness, Enable Continuous Improvement
  • "Without telemetry, you're flying blind"
  • Execution Monitoring (during, temporary) vs Operational Monitoring (after, permanent)
  • Execution tracks: Milestones, Budget/Resources, Blockers, Risk Indicators

3 Metric Categories

CategoryExamples
Security EffectivenessMTTD, false positives, ATT&CK coverage
Operational HealthUptime, latency, CPU
Business ValueCost savings, ROI, audit results
  • 3 Drivers of Change: Threat Landscape, Business Change, Technology Evolution
  • Feedback Loop: Collect → Analyze → Identify → Act → Validate (continuous)
  • Leading (predict: vuln scans, patching %, phishing clicks) vs Lagging (measure past: breaches, IR time, downtime)
  • Vanity (look good, no action: "5,427 rules") vs Actionable (inform decisions: "95% patched in 7-day SLA")
  • Test: "What action does this metric inform?" If none → vanity
  • 4-Step Baseline: Measure current → Set target (w/ interim) → Track progress → Validate
  • SMART: Specific, Measurable, Actionable, Relevant, Time-bound
  • Traceability: Goal → Implementation → Key Metrics

Review Cadence & Audience

  • Weekly (tactical/SOC) | Monthly (strategic/CISO) | Quarterly (comprehensive/cross-functional) | Annual (deep dive/board)
  • Technical=detail dashboards | Leadership=trend summaries | Executives=scorecards/biz language | Board=heat maps/risk posture
  • Virtuous Cycle: Design → Telemetry → Collect → Analyze → Adapt → Validate → repeat
  • Sustainable: Automate, Distribute ownership, Document, Review & prune
  • 5 Pitfalls: Measure everything, Perfect vs good, Ignore context, Static metrics, No actions defined

W12: Diffusion & Perturbation

  • 80% IT projects hit unexpected events, 60% scope creep
  • Scope Failure Type 1 (Foundational): entire domain wrong → full redesign
  • Scope Failure Type 2 (Drift): implementation drifted → targeted corrections
  • Scope Creep 3 impacts: Budget Overruns, Resource Exhaustion, Optimization Failure

Requirement Failures

  • Complete Misunderstanding (rare, full redesign)
  • Misinterpreted Prioritization (common, reprioritize)
  • Partial Coverage (most common, fix in implementation)
  • Requirements = collaborative dialogue, NOT checklist (Dr. Perez)

Recovery Framework (5 Steps)

  • Detect Signal → Classify Obstacle → Assess Severity → Determine Path → Execute & Validate

Decision Matrix 6 Factors

  • Issue Scope, Requirements Understanding, Security Outcome Impact, Dependency Effects, Stakeholder Alignment, Resource Requirements
  • Critical factors TRUMP others — if security outcomes can't be met → full iteration

Resources & Communication

  • Resource: Temporary (timeline impact, adjust/wait) vs Permanent (may need redesign, get risk acceptance IN WRITING)
  • Communication Breakdown 4 stages: Sporadic updates → Lose context → Support erodes → Project derailed
  • Communication failure = ROOT CAUSE; other failures = symptoms
  • 6 Elements of Deliberate Communication: formal plan, clear cadence, act as hub, transparency, document all, don't assume

PDCA & Foundations

  • PDCA (Shewhart-Deming): Plan → Do → Check → Act (repeats continuously)
  • 3 Foundations: Establish Review Cadence, Refine Metrics Over Time, Create Change Barometers
  • Executive Champion: reinforces with peers, provides air coverage, validates business case
  • Building case: use data (risk reduction, ROI), align with champion's own goals

W13: Maturity & Professional Excellence

  • Single Iteration (project, one at-bat) vs Sustainable Program (career, builds on lessons, never ends)
  • 2 Levels: Level 1 = Individual Design Optimization | Level 2 = Process Optimization (meta-level)
  • 3 Elements: Practice (iteration, tabletop exercises), Learning (measurement, metrics), Habit (self-reinforcing process)
  • Alignment: Temporal (match org rhythms: sprints, quarters) + Philosophical (cloud-first → cloud-native design)

Soft Skills & Wisdom

  • Active Listening: don't prepare Q3 while hearing Q2; stakeholders hold back because they don't know what's relevant
  • Multi-Disciplinary (Steve Oran, Intel): bring threat hunting + identity + ops + business together
  • Ego: Healthy confidence = about the work | Unhealthy ego = about you
  • Walk Away: validate first, but walking away early = good judgment. Implement stop-gaps, document why
  • Communication: over-communicate deliberately. Worst case of sharing = "I knew that." Worst case of not sharing = lost alignment
  • Process fit: Startup=lightweight | Mid-size=defined roles | Enterprise=formal governance
  • Empathy (Mark Simoes, Microsoft): look through others' eyes. CEO → compliance; ops → performance; users → usability

3 Great Architect Traits

  • Technical Foundation, Interpersonal Excellence, Strategic & Adaptive Thinking

5 Maturity Levels

  • Ad Hoc (reactive) → Repeatable (some structure) → Defined (standardized) → Managed (quantitative) → Optimizing (continuous improvement)
  • Politics: understand the landscape but stay neutral
  • Stay engaged through completion — strong start ≠ strong finish
  • Don't shirk preparation — shortcuts waste more time, lose defensibility
